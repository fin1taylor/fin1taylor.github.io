Bad Calculator
Android App
Project Overview
I first had the idea to make an calculator app that gives you the wrong answers while in math class. For Daily UI, I designed a calculator on day 4 and then I used that design to bring my idea to life.

Visit the Bad Calculator App on the Google Play Store
The Application
The Application has two modes: the Bad Calculator mode and the Normal Calculator mode. In the Bad Calculator mode, the answers will be slightly off the actual answer and in the Normal Calculator mode, the answers will be correct. There is also a Dark mode that simply changes the interface to dark colors.
The Bad Calculator doesn't just give you a random answer, it's smart! By default, it'll give you an answer that's 1 off the actual answer but if the actual answer is above 10 it'll be off by either 1 or 2. If your multiplying or dividing by 5 then the false answer will still be a multiple of 5. It'll do the same thing for 2 or any even number as well as 3 where the bad answers will be multiples of 2 and 3 respectfully. ‍

I also wanted to be able to use the calculator on an everyday basis so I decided to add a "normal calculator mode" where to answers are correct. Then I added a dark mode because I thought it looked nice.
The Process
While making the Bad Calculator App I tried different ways to program and design the application. I designed the first version of the application in android studio using Java. Then, for the second version of the app, I decided to try using a website called Thunkable. I first heard of this tool while helping to mentor young teens for a program called Technovation in which groups compete to create an application that benefits their community or society. I helped the teams from my school with the programming and design aspects of their apps and learned about Thunkable in the process. Thunkable is an amazing website that lets you code applications using blocks. So to learn the how the website worked I decided to redesign the Bad Calculator App. In retrospect, Thunkable is great for beginners or people who don't really know how to program. But it's also very limiting if you want to do more advanced things. The code that it creates is also not very optimised and much larger than it needs to be. If I were to create a third version of the application, i would probably code it in Android studio again so that it would be faster and more optimized.
